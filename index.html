Hardware

Conozcamos un poco sobre el Hardware

El hardware son todas las partes tangibles que forman un dispositivo electrónico, en este caso hablaremos de los computadores: Estos estan conformados por un chasis, procesador, tarjeta madre, memoria RAM, disco duro y perifericos E/S.

Conozcamos un poco sobre los componentes del Hardware

Hardware de procesamiento: Corresponde a la Unidad Central de Procesamiento o CPU, el centro de operaciones lógicas de la computadora, en donde se interpretan y ejecutan las tareas necesarias para el funcionamiento del resto de los componentes.

Hardware de almacenamiento: Hace referencia a todos los componentes cuya función es resguardar la información para que el usuario pueda acceder a ella en cualquier momento. El dispositivo principal en este caso es la memoria RAM (Random Access Memory), pero también está conformada por memorias secundarias, como los discos duros o las memorias SSD o USB.

Hardware gráfico: Está compuesto principalmente por las tarjetas gráficas que poseen memoria y CPU propia, y son las encargadas de interpretar y ejecutar las señales dedicadas a la construcción de imágenes. El hecho de que esta función no recaiga en la CPU tiene que ver con la eficiencia, ya que al liberar de esas funciones a la memoria principal, el sistema funciona de manera óptima.

Dispositivos periféricos: Es todo el hardware que permite que la información pueda ingresar a la computadora, o salir al exterior. Se subdividen en tres categorías.

Periféricos de entrada: son los que permiten el acceso de datos a la computadora. Por ejemplo, el teclado, el micrófono, la cámara web, etc.

Periféricos de salida: a través de ellos, el usuario puede extraer la información, como los escáneres, las impresoras, las consolas y los parlantes o altavoces.

Periféricos de entrada y salida o mixtos: son capaces de introducir o extraer la información. Las pantallas táctiles entran en esta categoría, al igual que los lectores de DVD o BlueRay y las memorias flash. Estas últimas son esencialmente memorias secundarias, pero se usan como periféricos mixtos.

Evolución del hardware

Desde la aparición de las primeras máquinas computacionales en la década de los 40 hasta la actualidad, la creación de hardware ha evolucionado para adaptarse a las nuevas tecnologías y usuarios.

De esta serie de cambios, se distinguen 4 generaciones de hardware:

- generación de hardware (1945-1956): uso de tubos de vacío en máquinas de cálculo.

- Segunda generación de hardware (1957-1963): los tubos al vacío fueron sustituidos por transistores.

- Tercera generación de hardware (1964-actualidad): creación de componentes basados en circuitos integrados impresos en una pastilla de silicio.

- Cuarta generación de hardware (futuro): todo hardware elaborado con nuevos materiales y formatos distintos al silicio, y que aún están en fase de investigación, diseño, desarrollo o implementación.

Tutorial de Git

¿Que es Git?

Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y realizar un seguimiento de los cambios en el código fuente a lo largo del tiempo.

Facilita la colaboración entre múltiples personas en un proyecto y permite revertir a versiones anteriores si es necesario.

Para crear un nuevo repositorio:

- Navega a la carpeta donde deseas crear el repositorio y ejecuta: git init

- Se crean directorios con mkdir "nombredelacarpeta"

- Se crean los archivos con el comando touch

- Para agregar archivos al repositorio git add "nombredelarchivo"

- O para agregar todos los archivos de la carpeta: git add .

- Para guardar los cambios realizados: git commit -m seguido de un mensaje que describa los cambios

- Para ver qué cambios se han realizado: git status

- Para revisar el historial de commits: git log

Crear y trabajar con ramas o branches:

Las ramas (o branches) en Git son una característica fundamental que permite a los desarrolladores trabajar en diferentes versiones de un proyecto al mismo tiempo. Cada rama es esencialmente una línea independiente de desarrollo en el mismo repositorio, lo que facilita la implementación de nuevas características, la corrección de errores o la experimentación sin afectar la rama principal (usualmente llamada main o master). Las ramas son una herramienta poderosa que te ayuda a organizar y gestionar tu trabajo en Git, facilitando la colaboración y el desarrollo ágil.

¿Para qué se utilizan los branches?

1.Desarrollo paralelo: Permiten que varios desarrolladores trabajen en diferentes características al mismo tiempo sin interferir entre sí.

2.Experimentación: Puedes probar nuevas ideas o enfoques en una rama separada sin afectar el código estable.

3.Mantenimiento: Facilitan la corrección de errores en una versión específica del código sin interrumpir el desarrollo de nuevas funciones.

4. Integración: Una vez que una funcionalidad ha sido completada y probada en una rama, puedes fusionarla de nuevo en la rama principal.

¿Como crear los branches?

- Para crear una nueva rama: git branch nombredelarama

- Para cambiar a una rama: git checkout nombredelarama

- Para crear una nueva e ingresar a ella: git checkout -b nombredelarama

- Cambia a la rama principal (usualmente 'main' o 'master'): git checkout master

* Comando merge: El comando merge en Git se utiliza para combinar dos ramas diferentes en una sola. Este comando permite integrar los cambios realizados en una rama (por ejemplo, una rama de características) a otra rama (generalmente la rama principal, como main o master).

- Fusiona la rama deseada con el comando merge: git merge nombredelarama

- Listar todas las ramas: git branch

- Eliminar una rama: git branch -d nombredelarama

Consejos para usar branches:

• Nombres descriptivos: Usa nombres que describan la funcionalidad o el propósito de la rama (por ejemplo, feature/login, bugfix/navbar).

• Mantén la rama principal limpia: Fusiona cambios solo cuando estén completamente probados y listos.

• Usa ramas para cada tarea: Crea una nueva rama para cada nueva funcionalidad o corrección de errores.

Agregar un repositorio remoto a GitHub:

Pasos para conectar tu repositorio local a GitHub

Crear un repositorio en GitHub:

- Inicia sesión en tu cuenta de GitHub.

- Haz clic en el botón "New" para crear un nuevo repositorio.

- Asigna un nombre a tu repositorio, elige si será público o privado y haz clic en “Create repository"

- Obtener la URL del repositorio:

- Una vez creado el repositorio, GitHub te proporcionará una URL. Puedes elegir entre HTTPS o SSH. La URL se verá algo así:

HTTPS: https://github.com/dianaed08/tutorial-Git.git

SSH: git@github.com:dianaed08/tutorial-Git.git

- Ahora en Git debes abrir tu terminal:

- Navega hasta el directorio de tu repositorio local. Usa el comando: 'cd'

- Agregar el repositorio remoto:

Usa el siguiente comando para agregar el repositorio remoto. Reemplaza 'nombre-remoto' (origin) y la URL de GitHub:

git remote add nombre-remoto URL GitHub

- Verificar que el repositorio remoto se ha agregado correctamente:

Para asegurarte de que el repositorio remoto se ha configurado correctamente, puedes usar el comando git remote -v, esto debería mostrarte las URLs del repositorio remoto para 'fetch' y 'push'.

- Si se equivocan como yo, pueden eliminarlos con el comando: git remote remove nombre-remoto

- Para subir tus cambios al repositorio remoto usa el siguiente comando para hacer un push:

git push -u origin master

Si la rama principal se llama main se debe cambiar el master por main.

La primera vez va a solicitar iniciar sesión en el navegador, debes darle los permisos y ahí si carga los cambios

Con estos pasos, tu repositorio local estará conectado al remoto en GitHub. ¡Ahora puedes colaborar y compartir tu código fácilmente!
